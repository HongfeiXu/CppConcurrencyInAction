# Chapter 3: Sharing Data Between Threads

## 3.1 共享数据的问题

### 3.1.1 条件竞争（Race Condition）

- **含义**：程序现象或结果依赖多个线程的**执行顺序**（谁先跑到哪一步）。
- **良性**：顺序怎么交错，结果仍符合预期——有「竞」但不碍事。
- **恶性**：交错导致**不变量**被破坏（典型如链表插入/删除做到一半被别的线程看到），数据或逻辑不再可靠；日常说的「并发写出问题」多指这类。
- **数据竞争**（C++ 标准术语）：对同一内存位置**无合法同步**的并发访问，且**至少一次写** → **未定义行为**（详见书/标准内存模型，如 §5.1.2）。属于标准单独钉死的一类并发错误。

### 3.1.2 避免恶性条件竞争

思路概览（书中列的三条路）：

- **加锁保护（本章主线）**
  用互斥等手段包住对共享数据的修改，使**只有当前修改线程**能看到不变量的「中间态」；对其他线程而言，要么还没改，要么已经改完。标准库里对应机制从 **3.2 互斥量** 开始讲。

- **无锁（lock-free）结构**
  **不**靠互斥锁，而用原子操作、CAS（compare-and-swap，比较并交换）、精心设计更新步骤等，让其他线程在任意时刻看到的结构都处于**合法不变量**之下（或配合重试直至成功）。难度高，极依赖内存模型与指令重排等细节，容易写错。

- **软件事务内存（software transactional memory，STM）**
  像数据库事务一样：把读写在事务日志里累积，再尝试**提交**；若期间数据被别的线程改过或事务失败，则**提交失败**、可重试。研究热点，**C++ 标准无直接 STM 支持**，本书不展开（历史上另有事务内存 TS（Technical Specification，ISO 技术规范），与日常工程关系不大）。
